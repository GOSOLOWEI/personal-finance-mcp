# 数据库设计文档

> 版本：v0.3.0  
> 创建日期：2026-03-11  
> 更新日期：2026-03-11  
> 数据库：PostgreSQL 16+  
> ORM：Drizzle ORM

---

## 一、设计原则

1. **软删除**：所有核心业务表均含 `deleted_at` 字段，物理删除前置于审计场景
2. **时间戳标准化**：所有表含 `created_at` / `updated_at`，使用 `TIMESTAMPTZ` 类型
3. **UUID 主键**：使用 `gen_random_uuid()` 生成，避免自增 ID 的信息泄露
4. **金额精度**：所有金额字段使用 `NUMERIC(15,2)`，避免浮点数精度问题
5. **审计可追溯**：所有写操作通过 `audit_logs` 表记录操作历史
6. **扩展性预留**：`user_id` 字段预留多用户扩展，当前阶段默认单用户

## 三、表结构详细设计

### 3.1 `users` — 用户表（预留多用户扩展）

```sql
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username      VARCHAR(50)  NOT NULL UNIQUE,
  email         VARCHAR(255) UNIQUE,
  created_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  deleted_at    TIMESTAMPTZ
);
```

> 当前阶段默认只有一个系统级用户，`user_id` 作为保留字段用于后续多用户扩展。

---

### 3.2 `accounts` — 账户表

```sql
CREATE TABLE accounts (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID         NOT NULL REFERENCES users(id),
  name            VARCHAR(50)  NOT NULL,
  type            VARCHAR(20)  NOT NULL,
  -- 枚举值: checking | savings | alipay | wechat | credit_card | investment | cash | other
  initial_balance NUMERIC(15,2) NOT NULL DEFAULT 0,
  currency        VARCHAR(10)  NOT NULL DEFAULT 'CNY',
  note            VARCHAR(200),
  is_active       BOOLEAN      NOT NULL DEFAULT TRUE,
  sort_order      INTEGER      NOT NULL DEFAULT 0,
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  deleted_at      TIMESTAMPTZ
);

CREATE INDEX idx_accounts_user_id ON accounts(user_id) WHERE deleted_at IS NULL;
```

### 3.3 `categories` — 分类表（两级树形结构）

```sql
CREATE TABLE categories (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID         NOT NULL REFERENCES users(id),
  name            VARCHAR(30)  NOT NULL,
  type            VARCHAR(10)  NOT NULL,  -- income | expense
  parent_id       UUID         REFERENCES categories(id),  -- NULL 表示一级分类
  -- 约束：parent_id 指向的分类的 parent_id 必须为 NULL（限制最多 2 级）
  group_label     VARCHAR(50),
  -- 大类群展示标签，仅用于界面分组，不参与分类节点
  -- 如：「必要生活」「健康成长」「品质社交」「收入」「金融往来」等
  icon            VARCHAR(10),
  color           VARCHAR(7),
  is_preset       BOOLEAN      NOT NULL DEFAULT FALSE,
  is_hidden       BOOLEAN      NOT NULL DEFAULT FALSE,
  count_in_stats  BOOLEAN      NOT NULL DEFAULT TRUE,
  -- false = 记录资金轨迹但不纳入真实收支统计（往来收入、债务管理类）
  sort_order      INTEGER      NOT NULL DEFAULT 0,
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  deleted_at      TIMESTAMPTZ  -- 仅自定义分类可软删除
);

CREATE INDEX idx_categories_user_type ON categories(user_id, type) WHERE deleted_at IS NULL;
CREATE INDEX idx_categories_parent ON categories(parent_id) WHERE deleted_at IS NULL;
```

### 3.4 `transactions` — 交易记录表（核心表）

```sql
CREATE TABLE transactions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID         NOT NULL REFERENCES users(id),
  type            VARCHAR(10)  NOT NULL,  -- income | expense | transfer
  amount          NUMERIC(15,2) NOT NULL CHECK (amount > 0),
  category_id     UUID         REFERENCES categories(id),
  account_id      UUID         NOT NULL REFERENCES accounts(id),
  to_account_id   UUID         REFERENCES accounts(id),    -- 仅 transfer 类型使用
  occurred_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW(),      -- 交易发生时间（精确到秒）
  note            VARCHAR(500),
  tag_labels      JSONB,
  -- 结构化标签，每个维度 0/1 个值，仅 expense 类型使用
  -- 格式: {"method":"线上","behavior":"日常消费","consumption_type":"必选消费","scale":"小额支出","purpose":"生存必需"}
  source_text     TEXT,                                     -- 批量自然语言录入时保留原始文本片段
  is_system       BOOLEAN      NOT NULL DEFAULT FALSE,      -- 系统自动生成（如余额调整）
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  deleted_at      TIMESTAMPTZ,
  deleted_reason  VARCHAR(200)
);

-- 核心查询索引
CREATE INDEX idx_transactions_user_date ON transactions(user_id, occurred_at DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_transactions_account ON transactions(account_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_transactions_category ON transactions(category_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_transactions_type ON transactions(user_id, type) WHERE deleted_at IS NULL;
CREATE INDEX idx_transactions_tag_labels ON transactions USING GIN(tag_labels) WHERE deleted_at IS NULL;

-- 备注全文搜索索引
CREATE INDEX idx_transactions_note_fts ON transactions 
  USING GIN(to_tsvector('simple', COALESCE(note, '') || ' ' || COALESCE(source_text, '')));
```

**v0.3 变更说明**：
- `date DATE` → `occurred_at TIMESTAMPTZ`：支持精确到秒的时间戳（v0.2）
- 新增 `source_text TEXT`：批量自然语言录入时保留原始文本片段（v0.2）
- `tags TEXT[]` → `tag_labels JSONB`：结构化多维度标签，替代自由文本标签数组（v0.3）

### 4.1 `v_account_balances` — 账户实时余额视图

```sql
CREATE VIEW v_account_balances AS
SELECT
  a.id,
  a.user_id,
  a.name,
  a.type,
  a.currency,
  a.color,
  a.initial_balance,
  a.initial_balance
    + COALESCE(SUM(CASE WHEN t.type = 'income' THEN t.amount ELSE 0 END), 0)
    - COALESCE(SUM(CASE WHEN t.type = 'expense' THEN t.amount ELSE 0 END), 0)
    + COALESCE(SUM(CASE WHEN t.type = 'transfer' AND t.to_account_id = a.id THEN t.amount ELSE 0 END), 0)
    - COALESCE(SUM(CASE WHEN t.type = 'transfer' AND t.account_id = a.id THEN t.amount ELSE 0 END), 0)
  AS current_balance
FROM accounts a
LEFT JOIN transactions t ON (t.account_id = a.id OR t.to_account_id = a.id)
  AND t.deleted_at IS NULL
WHERE a.deleted_at IS NULL
GROUP BY a.id;
```

### 4.2 `v_monthly_category_summary` — 月度分类汇总视图

```sql
CREATE VIEW v_monthly_category_summary AS
SELECT
  t.user_id,
  TO_CHAR(t.occurred_at AT TIME ZONE 'Asia/Shanghai', 'YYYY-MM') AS year_month,
  t.type,
  t.category_id,
  c.name        AS category_name,
  c.parent_id,
  p.name        AS parent_category_name,
  COUNT(*)      AS transaction_count,
  SUM(t.amount) AS total_amount
FROM transactions t
JOIN categories c ON c.id = t.category_id
LEFT JOIN categories p ON p.id = c.parent_id
WHERE t.deleted_at IS NULL
GROUP BY t.user_id, year_month, t.type, t.category_id, c.name, c.parent_id, p.name;
```

### 4.3 `v_monthly_amortized_summary` — 含分摊的月度汇总视图

```sql
-- 计算某月应计分摊金额（不含已取消的分摊计划）

SELECT
  a.user_id,
  -- 生成分摊月序列
  TO_CHAR(
    (TO_DATE(a.start_month, 'YYYY-MM') + (gs.n - 1) * INTERVAL '1 month'),
    'YYYY-MM'
  ) AS year_month,
  a.id AS amortization_id,
  a.transaction_id,
  -- 最后一个月补足差额
  CASE
    WHEN gs.n = a.total_months THEN a.total_amount - a.monthly_amount * (a.total_months - 1)
    ELSE a.monthly_amount
  END AS amortized_amount,
  t.category_id
FROM amortizations a
JOIN transactions t ON t.id = a.transaction_id
CROSS JOIN generate_series(1, a.total_months) AS gs(n)
WHERE a.status != 'cancelled';
```

## 六、迁移策略

1. 使用 **Drizzle Kit** 管理数据库迁移，迁移文件存放于 `src/db/migrations/`
2. 每次 Schema 变更必须生成对应的迁移 SQL 文件，禁止直接手动修改生产数据库
3. 迁移命令：`npm run db:migrate`
4. 回滚策略：每个迁移文件需包含回滚 SQL（`down` 方向）

```bash
# 常用命令
npm run db:generate   # 生成迁移文件
npm run db:migrate    # 执行迁移
npm run db:studio     # 打开 Drizzle Studio 可视化界面
npm run db:seed       # 插入预设分类种子数据
```
