# 测试用例文档

> 版本：v1.0.0  
> 创建日期：2026-03-11  
> 说明：涵盖所有 39 个 MCP Tools 的功能测试、边界测试、异常测试

---

## 测试准备

### 前置条件

在开始测试前，确保已完成初始化：

```bash
cd code
npm run db:migrate   # 创建所有表
npm run db:seed      # 插入预设分类和标签
npm run dev          # 启动 MCP Server
```

### 测试账户数据

以下测试用例假设已创建如下账户（通过 `record_account` Tool 创建）：

| 变量名 | 账户名 | 类型 | 初始余额 |
|---|---|---|---|
| `ACCOUNT_ALIPAY` | 支付宝 | alipay | 5000.00 |
| `ACCOUNT_WECHAT` | 微信钱包 | wechat | 2000.00 |
| `ACCOUNT_BANK` | 招商银行储蓄卡 | checking | 30000.00 |
| `ACCOUNT_CREDIT` | 招商银行信用卡 | credit_card | 0.00 |
| `ACCOUNT_CASH` | 现金 | cash | 500.00 |

### TC-TXN-002：预算预警触发

```json
  "account_id": "{{ACCOUNT_ALIPAY}}",
```

```json
// 步骤 2：再录入 50 元（累计 250 元，83.3% > 80%）
{
  "type": "expense",
  "amount": 50.00,

  "account_id": "{{ACCOUNT_ALIPAY}}",
  "note": "外卖晚餐"
}
// 预期响应：
// - budget_warning.status = "warning"
// - budget_warning.used_percent ≈ 83
```

```json
// 步骤 3：再录入 100 元（累计 350 元，超支）
```

### TC-ERR-002：常见错误码覆盖

| 错误码 | 触发场景 | 验证 Tool |
|---|---|---|
| `VALIDATION_ERROR` | 参数格式错误、缺少必填字段 | 所有 Tool |
| `NOT_FOUND` | 指定 ID 不存在 | update/delete 类 Tool |
| `CONFLICT` | 余额无需校正；恢复未删除记录 | adjust_account_balance, restore |
| `CATEGORY_DEPTH_EXCEEDED` | 尝试创建三级分类 | record_category |
| `PRESET_CATEGORY_DELETE` | 尝试删除预设分类 | delete_category |
| `ACCOUNT_HAS_TRANSACTIONS` | 删除有交易的账户/分类 | delete_account, delete_category |
| `INVALID_TRANSFER` | transfer 类型缺少 to_account_id | record_transaction |
| `AMORTIZATION_MONTHS_TOO_FEW` | 分摊月数 < 2 | record_amortization |
| `AMORTIZATION_NOT_EXPENSE` | 对非支出交易创建分摊 | record_amortization |
