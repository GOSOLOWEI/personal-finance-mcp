# 开发路线图

> 版本：v0.2.0  
> 创建日期：2026-03-11  
> 更新日期：2026-03-11

---

## 一、总体开发策略

采用"**全量设计 → 分阶段交付**"策略：

- 一次性完成完整功能文档设计（避免后期返工）
- 分 4 个阶段逐步交付可用版本
- 每个阶段结束后均可独立运行和使用
- 后续阶段在前阶段基础上增量扩展

---

## 二、开发阶段划分

### 阶段一：基础骨架（Week 1-2）

**目标**：可运行的 MCP 服务，具备最核心的记账和查询能力

**交付内容**：

```
✅ 项目初始化
   - TypeScript + Node.js 项目脚手架
   - Drizzle ORM 配置 + PG 连接
   - MCP SDK 接入，基础 Server 可运行
   - .env 配置模板
   - ESLint + Prettier 规范

✅ 数据库
   - 所有表的初始 migration
     （users / accounts / categories / transactions / budgets / subscriptions / amortizations）
   - 预设分类种子数据插入（含二级分类）
   - Drizzle Studio 可视化

✅ 账户管理（5 Tools 全部完成）
   - record_account
   - update_account
   - delete_account
   - list_accounts（含净资产汇总，替代原 get_net_worth）
   - adjust_account_balance

✅ 交易记录（核心 4 Tools）
   - record_transaction（含账户余额联动更新，occurred_at 支持秒级时间戳）
   - update_transaction
   - delete_transaction（含余额回滚）
   - list_transactions（多条件过滤 + 分页 + 单条 ID 查询）

✅ 分类管理（3 Tools）
   - list_categories（返回二级分类树）
   - record_category（含二级限制校验）
   - update_category（含 hidden 参数，替代原 toggle_category_visibility）
```

**阶段一验收标准**：
- [ ] 能通过 Claude Desktop 自然语言记录一笔消费（时间精确到秒）
- [ ] 能查询本月所有支出列表
- [ ] 账户余额在记账后正确更新
- [ ] `list_accounts` 响应中净资产汇总数字正确

✅ 订阅管理（3 Tools 全部完成）

**阶段二验收标准**：
- [ ] 输入一段文字「今天买了咖啡18元，午餐35元，打车12元」，自动解析为 3 条记录

## 三、后续迭代方向（v2.0 规划）

> 以下功能在 v1.0 完成后根据实际使用反馈决定是否开发

| 功能 | 优先级 | 说明 |
|---|---|---|
| 多用户支持 | 高 | 家庭成员共享，数据严格隔离 |
| CSV 批量导入 | 高 | 支持导入支付宝/微信账单 CSV |
| 储蓄目标 | 中 | 按需重新引入，当前版本暂缓 |
| 定期任务自动化 | 中 | 每月自动生成净资产快照、预算结转、订阅扣费提醒 |
| 接入实时行情 API | 中 | 投资账户自动更新估值 |
| 数据导出 | 中 | 导出为 Excel/CSV |
| WebSocket 通知 | 低 | 实时预算预警推送 |
| 多货币支持 | 低 | 汇率转换 |

## 四、技术债与约束说明

| 约束项 | 说明 |
|---|---|
| AI 分类推荐 | `record_transaction` 不再有独立的 `suggest_category` Tool；AI 客户端根据描述自行推理分类后传入 `category_id` |
| AI 自动打标签 | `suggested_tags` 由 Service 层基于规则计算返回，AI 客户端可选择接受后在下次写入时带上 |
| 批量录入解析 | `batch_record_transactions` 的 NL 解析依赖 AI 客户端（Claude/GPT），MCP 服务端仅做最终数据落库 |
| 投资估值 | 不接入实时行情，需用户手动更新市值 |
| 定期任务 | v1.0 不含定时任务（如月度快照、订阅扣费自动记账），依赖用户手动调用 Tool 触发 |
| 并发安全 | 当前单用户场景下连接池默认 2-10，多用户版本需重新评估 |
| 分摊计算精度 | 每月分摊金额保留 2 位小数，最后一个月自动补足差额 |
