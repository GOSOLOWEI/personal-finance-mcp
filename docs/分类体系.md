# 个人收支二级分类体系（最终版）

> 版本：v1.0  
> 更新日期：2026-03-11  
> 状态：已确认，可用于数据库种子数据

---

## 一、设计决策说明

| 决策点 | 结论 |
|---|---|
| 层级结构 | 数据库两级（一级分类 + 二级分类）；大类群仅作 `group_label` 字段，不参与分类节点 |
| 🚫 不计入的技术实现 | **方案 C 混合**：账户间资金转移用 `transfer` 交易类型；偶发往来收入用 `income` + `count_in_stats=false` |
| 旅行 | 独立一级分类，机票/住宿/餐饮/门票统一归口 |
| 订阅费 | 散入各语义分类（Netflix→电影演出；ChatGPT→工具软件；话费→手机话费） |
| 房贷 | 全额计入支出（不拆分利息/本金） |
| 默认隐藏 | 不通用的分类（有车/有孩子/有宠物/往来债务类）默认 `is_hidden=true`，按需开启 |

---

## 二、统计标识规则

| 标识 | 含义 | 技术实现 |
|---|---|---|
| ✅ 计入 | 参与真实收支统计 | `category.count_in_stats = true`（默认） |
| 🚫 不计入 | 记录资金轨迹，不污染收支统计 | 账户间转移用 `transfer` 类型；偶发往来用 `count_in_stats = false` |

## 六、财务分析指标

| 指标 | 计算公式 | 含义 |
|---|---|---|
| **真实可支配收入** | = income[count_in_stats=true] - expense[count_in_stats=true] | 你真正「花掉」的钱和「剩下」的钱 |
| **总资金净流量** | = 所有 income - 所有 expense（含 transfer） | 银行账户里实际金额的增减 |
| **真实储蓄率** | = 真实可支配收入 / income[count_in_stats=true] | 真实的储蓄/结余能力 |

### 示例

本月：工资 10000 ✅ / 房租 3000 ✅ / 基金定投 2000（transfer）/ 还信用卡 4000（transfer）/ 收同事垫付 500 🚫

- **真实可支配收入**：10000 - 3000 = **7000 元**
- **总资金净流量**：+10000 - 3000 - 2000 - 4000 + 500 = **+1500 元**（账户余额净增）
- **真实储蓄率**：7000 ÷ 10000 = **70%**
